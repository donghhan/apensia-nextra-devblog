# 사용된 툴 및 라이브러리

## API 서버

### Node.js

웹 및 모바일 어플리케이션을 위한 API 서버의 경우 Node.js (좀 더 정확히 말하자면 타입스크립트를 곁들인 Express.js)를 사용하기로 하였다.

#### Django VS Node.js

필자가 즐겨 사용하던 Django와 Node.js를 비교해보았다. 이번 프로젝트는 단순히 오래 사용해와서 익숙하기 때문이라는 단편적인 이유만으로 속단을 내리면 안 될 것 같았다 (비교는 단순히 상대적인 것이다).

| 비교      | Django                                                    | Node.js (Express.js)                                   |
| --------- | --------------------------------------------------------- | ------------------------------------------------------ |
| 아키텍쳐  | <center>MVT</center>                                      | <center>이벤트 기반</center>                           |
| 보안성    | <center>높음</center>                                     | <center>낮음</center>                                  |
| 확장성    | <center>높은 트래픽과 큰 데이터를 다루는 데 용이</center> | <center>많은 양의 HTTP 요청을 처리하는데 용이</center> |
| 개발 속도 | <center>빠름</center>                                     | <center>느림</center>                                  |

Django의 경우

1. MVT 아키텍쳐를 채택하고 있다. 즉, 데이터를 다루는 모델(M)과 데이터와 비즈니스 로직을 가교 역할을 하는 뷰(V), 그리고 뷰에 의해서 만들어져 결과물을 유저에게 보여주는 템플릿(T)으로 구성되어 있다.
2. Django 자체로 보안성을 강화해주는 장치들 (예: SQL 인젝션, CSRF)
