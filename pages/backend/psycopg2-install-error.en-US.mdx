## Analysis

The core part seemed to be Clang, which is one of the macOS's Command Line Tools Packages.
Command Line Tools Package is the small package that can be downloaded separately from Xcode that allows us to do development with command line.[^1]

`xcode-select: Failed to locate 'clang', requesting installation of command line developer tools.`

## Solutions

I analyzed the problem into two ways:

1. It may not work in M1 chip &rarr; Let's use substitute!
2. Problem in Command Line Developer Tools

<br/>

### Attempt1 &rarr; 😓 Fail

I followed console's hint: using <a href="https://pypi.org/project/psycopg2-binary/" target="_blank">psycopg2-binary</a> package.

But still I got same error as above. So I realized that this has nothing to do with M1 chip or whatsoever.

<br/>

### Attempt2 &rarr; 😊 Success

I found one <a href="https://apple.stackexchange.com/questions/415563/repeatedly-being-asked-to-install-command-line-tools/446563#446563" target="_blank">result</a> on Stack Overflow and could solve the problem with the following command.

`sudo xcode-select -switch /Library/Developer/CommandLineTools`

And I could find another thing <a href="https://apple.stackexchange.com/a/446624" target="_blank">at the very below reply</a> that there has been issue in downloading XCode CLI tools 14.0 after updating M1 chip's Monterey 12.6 version.
This macOS updates always make me tired...

~~Then how could I have been able to install other packages such as Django and Pillow?~~
