# 웹 접근성에도 차별 금지법이 있다고?

웹 퍼블리싱에 있어 웹 표준, 웹 호환성 말고도 웹 접근성이라는 개념이 있다.
언뜻 보면 비슷비슷해 보이지만 역시 다른 개념을 가지고 있다.

## "접근성"이란?

접근성에 대한 사전적인 의미는 <ins>**_"사용자가 어떠한 제품이나 서비스 등에 접근해 편리하게 이용할 수 있는 정도"_**</ins>이다.
그렇다면 웹에서의 접근성이란 <ins>**_"모든 사용자가 모든 기기에서 어떠한 제약없이 웹에 접근할 수 있는 정도"_**</ins>로 해석될 수 있겠다.

> 웹 접근성을 지켜야 하는 이유는 역시 다름아닌 법과 연결되어 있기 때문이다.
> 한국을 기준으로 얘기하자면 2013년 4월 11일부터 소위 <a href="https://law.go.kr/%EB%B2%95%EB%A0%B9/%EC%9E%A5%EC%95%A0%EC%9D%B8%EC%B0%A8%EB%B3%84%EA%B8%88%EC%A7%80%EB%B0%8F%EA%B6%8C%EB%A6%AC%EA%B5%AC%EC%A0%9C%EB%93%B1%EC%97%90%EA%B4%80%ED%95%9C%EB%B2%95%EB%A5%A0" target="_blank">장애인차별금지법(줄여서 "장차법")</a>이라 불리우는 법으로 인해
> 장애인들도 비장애인들과 동등하게 전자정보 및 비전자정보를 사용할 수 있도록 해야 한다는 내용이 포함되어 있다.
> 악의로 인정이 될 경우 3년 이하의 징역이나 3천 만원 이하의 벌금이 부과될 수도 있다...

### "장애"의 종류

굉장히 모호하게 들리는 "장애"라는 단어는 여러 가지를 내포할 수도 있다. ~~늘 그래왔듯이 법은 우리에게 친절하지 않다.😤~~

1. **저시력 시각 장애**
   : 안경이나 렌즈 등 현대 의학으로는 해결할 수 없는 시력을 가진 장애를 뜻한다. 화면이 뿌옇게 보이기 때문에 화면의 내용을 잘 인식할 수 없게 된다.
   &rarr; 더욱 크게 볼 수 있도록 <ins>_화면 확대 기능_</ins>을 사용한다.

2. **색약/색맹**
   : 색상을 분별하는 어려움을 일컫는다.
   &rarr; 더욱 선명하게 웹 페이지를 사용할 수 있도록 <ins>_고대비_</ins>를 사용한다.

3. **전맹 시각 장애**
   : 아예 시력이 없어 볼 수 없는 장애를 의미한다.
   &rarr; 프로그램 실행 후에 웹에 접근하면 웹 페이지를 읽어주는 청각적으로 도움을 줄 수 있는 <ins>**스크린 리더 장치**</ins>를 이용하거나 촉각적으로 도움을 줄 수 있는 <ins>**점자 정보 단말기**</ins>를 사용한다.

4. **중증 운동 장애**
   : 손 혹은 팔을 이용하지 못하고 목만 움직일 수 있는 장애를 의미한다.
   &rarr; 한 손 사용자용 키보드, 트랙볼 마우스 등 <ins>각종 보조기기</ins>를 사용한다.

5. **청각 장애**
   : 글자 그대로 들을 수 없는 장애를 의미한다.
   &rarr; <ins>안내 문구나 자막같은</ins> 화면의 글씨들을 읽는다.

6. **맥 장애**
   : 윈도우에서만 접근이 가능해 맥 사용자는 접근할 수 없는 장애를 일컫는다. ~~아런 것도 장애가 될 수 있다니...😳~~

7. **느린 인터넷**
   : 더 이상 자세한 설명은 생략하겠다...

## 웹 접근성에 대한 지침

그렇다면 웹 접근성에 대한 지침에는 어떤 것들이 있는지 프론트엔드 개발 단에서 해결할 수 있는 부분들 중 몇 가지를 살펴보도록 하겠다.

### 적절한 대체 텍스트 `alt` 제공

텍스트가 아닌 모든 콘텐츠는 그 의미나 용도를 알 수 있도록 대체 텍스트를 제공해야 한다.
가장 대표적인 예로는 `img` 태그 안에 있는 <a href="https://developer.mozilla.org/ko/docs/Web/HTML/Element/img#attr-alt" target="_blank"><code>alt</code></a> 속성이 되겠다.
이 `alt` 대체 텍스트가 있게 되면 스크린 리더가 이를 읽어주게 된다.

```html mark=1[67:90]
<img src="https://www.rnib.org.uk/sites/default/files/CD_0_0.jpg" alt="government-support">
```

만일 대체 텍스트가 너무 긴 경우에는 `alt`에 빈 값을 제공하는 대신 `p` 태그와 같이 마크업으로 대체 텍스트를 제공할 수 있다.

```html
<img src="dog.jpg" alt="" />
<p>
    This is my beautiful dog! What an awesome and cute boy he is!
</p>
```

### 색에 무관한 콘텐츠 인식

색을 구별하지 못하더라도 콘텐츠 인식에 문제가 없어야 한다.

이는 페이지네이션이나 링크 등을 단순히 색으로만 사용하는 것이 아니라 굵기 혹은 배경 색상 등을 적절하게 넣어서 색맹인 사람들도 쉽게 콘텐츠를 인식할 수 있도록 하면 된다.

<br/>

### 명확한 지시 사항 제공

지시 사항을 특정 단일 감각에만 의존하는 방향으로 제공하면 안 되고, 여러 다른 감각을 통해서도 지시 사항을 이해하는데 문제가 없어야 한다.

예를 들어, 회원가입 란의 필수 입력 필드 앞에 단순히 <ins>"\*"</ins>를 넣는 것보다는 상단에 <ins>"\*는 필수 입력 사항입니다."</ins> 등의 설명 문구를 집어넣으면 된다.

<br/>

### 자동 재생 금지

말 그대로 콘텐츠가 제공하는 모든 소리는 자동으로 재생되어선 안 된다.[^1] 웹 사이트 이용자가 방해를 받을 수 있기 때문이다. ~~혹은 당황스러움을 회피하기 위하여...~~
